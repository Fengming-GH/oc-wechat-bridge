## 1. 架构总览

### 1.1 数据流

```
┌─ 微信用户 ──────────────────────────────────────────────────┐
│  发送消息 → iLink API (腾讯服务器)                              │
│                                                              │
│                  ↓ POST /ilink/bot/getupdates (长轮询 35s)      │
│                                                              │
│  ┌─ OpenCode 进程 ──────────────────────────────────────────┐ │
│  │                                                          │ │
│  │  Plugin: @fengming-gh/oc-wechat-bridge                    │ │
│  │                                                          │ │
│  │  [长轮询协程]                                             │ │
│  │    while (!aborted)                                      │ │
│  │      await getupdates(syncBuffer, 35s)  ◄── HTTP hold ──── │
│  │      for each msg:                                       │ │
│  │        client.session.prompt({ path, body }) ──→ OC Server │ │
│  │                                                          │ │
│  │  [事件钩子] ←── OC Server SSE 事件流 ───────────────────── │ │
│  │    session.idle → sendText(最终回复)  ──→ iLink API        │ │
│  │    session.created → 更新缓存                               │ │
│  │    permission.ask → sendText(审批请求) ──→ iLink API       │ │
│  │                                                          │ │
│  └──────────────────────────────────────────────────────────┘ │
│                              ↑                                │
│                  POST /ilink/bot/sendmessage                   │
│                                                              │
│  收到回复 ← iLink API (腾讯服务器)                               │
└──────────────────────────────────────────────────────────────┘
```

### 3.4 Section 3: Utility Helpers

```typescript
// ============================================================
// Section 3:  Utility Helpers
// ============================================================

/** 返回北京时间字符串，格式 "2026-06-23 14:30:45" */
function bjNow(): string {
  return new Intl.DateTimeFormat("zh-CN", {
    timeZone: "Asia/Shanghai",
    year: "numeric", month: "2-digit", day: "2-digit",
    hour: "2-digit", minute: "2-digit", second: "2-digit",
    hour12: false,
  }).format(new Date()).replace(/\//g, "-")
}

/** 追加日志到 <插件目录>/log/wechat-bridge.log */
function log(level: string, msg: string) {
  try {
    appendFileSync(LOG_PATH, `${bjNow()} [${level}] ${msg}\n`, "utf-8")
  } catch { /* best effort */ }
}

/** sessionID → 可读标题，找不到则返回 "?<前8位>" */
function t(sid: string): string {
  return sidTitle.get(sid) ?? "?" + sid.slice(0, 8)
}

/** Promise 形式的 sleep */
function sleep(ms: number): Promise<void> {
  return new Promise(r => setTimeout(r, ms))
}

/** 生成一次性审批码（4 位大写字母数字） */
function buildOneTimeCode(): string {
  const chars = "ABCDEFGHJKLMNPQRSTUVWXYZ23456789"
  let code = ""
  for (let i = 0; i < 4; i++) code += chars[Math.floor(Math.random() * chars.length)]
  return code
}
```

**与原 bridge 的差异**：

- `buildOneTimeCode()` 从 `bridge-utils.js` 抽取

```typescript
/** AES-128-ECB 加密（微信 CDN 使用） */
function encryptAesEcb(plaintext: Buffer, key: Buffer): Buffer {
  const cipher = createCipheriv("aes-128-ecb", key, null)
  return Buffer.concat([cipher.update(plaintext), cipher.final()])
}

/** AES-128-ECB 解密 */
function decryptAesEcb(ciphertext: Buffer, key: Buffer): Buffer {
  const decipher = createDecipheriv("aes-128-ecb", key, null)
  return Buffer.concat([decipher.update(ciphertext), decipher.final()])
}

/** CDN 加密后的实际上传大小（向上取整到 16 字节倍数，+1 字节填充） */
function aesEcbPaddedSize(plaintextSize: number): number {
  return Math.ceil((plaintextSize + 1) / 16) * 16
}

/** 生成每请求唯一的 X-WECHAT-UIN 头 */
function randomWechatUin(): string {
  const uint32 = randomBytes(4).readUInt32BE(0)
  return Buffer.from(String(uint32), "utf-8").toString("base64")
}

/** 解码入站媒体的 AES key（支持 hex 或 base64 编码的 16 字节密钥） */
function decodeInboundMediaAesKey(value: string): Buffer {
  const trimmed = value.trim()
  // hex 格式 (32 hex chars = 16 bytes)
  if (/^[a-f0-9]{32}$/i.test(trimmed)) return Buffer.from(trimmed, "hex")
  // base64 格式
  const decoded = Buffer.from(trimmed, "base64")
  if (decoded.length === 16) return decoded
  // base64 解码后是 hex 字符串
  const decodedText = decoded.toString("utf8").trim()
  if (/^[a-f0-9]{32}$/i.test(decodedText)) return Buffer.from(decodedText, "hex")
  throw new Error("Unsupported inbound media aes key format: " + value.slice(0, 20))
}

/** 解密入站 CDN 媒体载荷 */
function decryptInboundMediaPayload(ciphertext: Buffer, aesKey: string): Buffer {
  return decryptAesEcb(ciphertext, decodeInboundMediaAesKey(aesKey))
}

/** 加密出站消息的 AES key（hex → base64 编码） */
function encodeMessageAesKey(aesKey: Buffer): string {
  return Buffer.from(aesKey.toString("hex")).toString("base64")
}

/** CDN 下载 URL 构造 */
function buildCdnDownloadUrl(encryptQueryParam: string): string {
  return `${CDN_BASE_URL}/download?encrypted_query_param=${encodeURIComponent(encryptQueryParam)}`
}

/** CDN 上传 URL 构造 */
function buildCdnUploadUrl(uploadParam: string, filekey: string): string {
  return `${CDN_BASE_URL}/upload?encrypted_query_param=${encodeURIComponent(uploadParam)}&filekey=${encodeURIComponent(filekey)}`
}
```

```typescript
async function updateSessionIcon(client: any, sid: string, status: "normal" | "degraded" | "processing" | "offline") {
  try {
    let title = sidTitle.get(sid)
    if (!title) return
    const base = stripIconPrefix(title)
    const iconMap: Record<string, string> = {
      normal: WECHAT_ICON,
      degraded: WECHAT_ICON_DEGRADED,
      processing: WECHAT_ICON_PROCESSING,
      offline: WECHAT_ICON_OFFLINE,

    if (newTitle !== title) {

      sidTitle.set(sid, newTitle)
    }

}

/** 各 sid 的发送失败累计次数（连续失败达到阈值后图标变灰） */
const sendFailCount = new Map<string, number>()
const SEND_FAIL_THRESHOLD = 3

/** 记录发送成功/失败，失败达阈值时自动降级图标 */
function recordSendResult(sid: string | null, success: boolean, client: any) {
  if (!sid) return
  const current = sendFailCount.get(sid) ?? 0
  if (success) {
    sendFailCount.set(sid, 0)
    updateSessionIcon(client, sid, "normal")
  } else {
    const next = current + 1
    sendFailCount.set(sid, next)
    if (next >= SEND_FAIL_THRESHOLD) {
      updateSessionIcon(client, sid, "degraded")
    }
  }
}

/** 获取或创建 wechat sender 对应的 OC session */
async function getOrCreateSession(client: any, wechatId: string, worktree: string): Promise<string> {
  // 1. 已有映射（内存 + 磁盘双重确认）
  const existing = wechatSid.get(wechatId)
  if (existing && sidTitle.has(existing)) {
    // 恢复为正常状态（以防之前是降级/处理中状态）
    updateSessionIcon(client, existing, "normal")
    return existing
  }

  // 2. 创建新 session
  try {
    const title = wechatTitle(wechatId)
    const resp: any = await client.session.create({
      body: { title },
    })
    const sid = resp.id ?? resp.sessionID ?? resp.data?.id
    if (sid) {
      wechatSid.set(wechatId, sid)
      sidTitle.set(sid, title)
      saveSessionMapping(worktree)  // 持久化
      await updateSessionIcon(client, sid, "normal")
      log("SESSION", `created [${t(sid)}] for wechat:${wechatId.slice(0, 8)}`)
      return sid
    }
  } catch (err) {
    log("SESSION_CREATE_FAIL", `${err}`)
  }

  // 3. 保底：取第一个可用 session
  try {
    const resp: any = await client.session.list()
    const all: Session[] = Array.isArray(resp) ? resp : resp.data ?? []
    const first = all.find((s: Session) => !s.parentID)
    if (first) {
      wechatSid.set(wechatId, first.id)
      sidTitle.set(first.id, `${WECHAT_ICON}${first.title}`)
      await updateSessionIcon(client, first.id, "normal")
      saveSessionMapping(worktree)  // 持久化
      return first.id
    }
  } catch { /* best effort */ }
  throw new Error("No available session")
}

  }
  return null
}

        wechatSid.set(wx, sid)
      }
    }
    if (wechatSid.size > 0) {
      log("MAP", `restored ${wechatSid.size} session mappings from ${fp}`)
    }
  } catch (e) {
    log("MAP_LOAD_ERR", `${e}`)
  }
}

  }

      if (full !== worktree && existsSync(join(full, ".opencode"))) {

      }
    }
  } catch { }
  return dirs
}
```

**设计要点**：
- 每个微信用户绑定一个 OC session（通过 `wechatSid` Map）
- session 创建延迟到第一条消息到达时（`getOrCreateSession`）
- `findWechatSender` 用于钩子中（如 `session.idle`）找到正确的微信发送者
- 映射持久化到 `{worktree}/.opencode/plugins/wechat-bridge/session-map.json`，插件重启后自动恢复
- 映射变更后立即写盘（原子写入 tmp + rename），保证崩溃不丢失
- **零侵入**：未绑定微信时插件完全透明，OC 正常使用不受任何影响
- **微信图标标记**：会话标题前缀作为状态指示灯，不同状态不同图标：

  | 图标 | 含义 | 触发 |
  |:----:|:-----|:-----|
  | `📱` | 正常 | 默认状态，发送成功时恢复 |
  | `💬` | AI 处理中 | 消息注入 OC 后、`session.idle` 前 |
  | `📵` | 发送降级 | `sendText`/`sendImage` 连续失败 >= 3 次 |
  | `🔴` | 登录过期 | WeChat session timed out / 401 |

  状态切换通过 `updateSessionIcon(client, sid, status)` 完成，`session.deleted` 时清理。

```typescript
// ============================================================
// Section 7:  WeChat HTTP Transport

/** 基础 HTTP POST 请求（含超时、鉴权、错误处理） */
async function apiFetch(
  endpoint: string,
  body: object,
  token: string,
  timeoutMs: number,
  signal?: AbortSignal,
): Promise<string> {
  const base = BASE_URL.endsWith("/") ? BASE_URL : BASE_URL + "/"
  const jsonBody = JSON.stringify(body)

  const controller = new AbortController()
  const timer = setTimeout(() => controller.abort(), timeoutMs)
  if (signal) {
    signal.addEventListener("abort", () => controller.abort(), { once: true })
  }

  try {
    const res = await fetch(`${base}${endpoint}`, {
      method: "POST",
      headers: {
        "Content-Type": "application/json",
        AuthorizationType: "ilink_bot_token",
        "X-WECHAT-UIN": randomWechatUin(),
        "Content-Length": String(Buffer.byteLength(jsonBody, "utf-8")),
        Authorization: `Bearer ${token}`,
      },
      body: jsonBody,
      signal: controller.signal,
    })
    clearTimeout(timer)
    const text = await res.text()
    if (!res.ok) throw new Error(`HTTP ${res.status}: ${text.slice(0, 200)}`)
    if (!text.trim()) return text

    // 解析并检查 API 错误码
    // 注意：如果响应体不是 JSON（如网关返回的纯文本错误），JSON.parse 会抛异常，
    // 该异常被外层 catch 捕获并按 network error 重试。
    const parsed = JSON.parse(text)
    const ret = parsed.ret ?? 0
    const errcode = parsed.errcode ?? 0
    const errmsg = parsed.errmsg ?? ""

    if (ret !== 0 || errcode !== 0) {
      log("API_ERR", `${endpoint} ret=${ret} errcode=${errcode} errmsg=${errmsg}`)
      // session 超时
      if (errcode === -14 && /session timeout/i.test(errmsg)) {
        throw new Error("WeChat session timed out")
      }
      // context_token 过期
      if (ret === -2) {
        throw new Error("Context token stale")
      }
      throw new Error(`API error: ${endpoint} ret=${ret} errcode=${errcode}`)
    }
    return text
  } catch (err) {
    clearTimeout(timer)
    throw err
  }
}

/** 长轮询获取新消息 */
async function getUpdates(account: WechatCredentials, timeoutMs: number, signal?: AbortSignal): Promise<{
  msgs: any[]
  get_updates_buf: string | null
}> {
  const raw = await apiFetch(
    "ilink/bot/getupdates",
    { get_updates_buf: syncBuffer, base_info: { channel_version: CHANNEL_VERSION } },
    account.token,
    timeoutMs,
    signal,
  )
  if (!raw.trim()) return { msgs: [], get_updates_buf: null }
  const parsed = JSON.parse(raw)
  const newBuf = parsed.get_updates_buf ?? null
  if (newBuf) {
    syncBuffer = newBuf
    saveSyncBuffer(syncBuffer)
  }
  return {
    msgs: parsed.msgs ?? [],
    get_updates_buf: newBuf,
  }
}

/** 发送文本消息到微信（含失败计数 + 图标状态降级） */
async function sendText(
  account: WechatCredentials,
  recipientId: string,
  text: string,
  contextToken?: string,
  client?: any,
) {
  const trimmed = text.trim()
  if (!trimmed) return

  let token = contextToken
  if (!token) token = contextTokens.get(recipientId)
  if (!token) {
    log("SEND_SKIP", `no context token for ${recipientId.slice(0, 16)}`)
    return
  }

  try {
    await apiFetch(
      "ilink/bot/sendmessage",
      {
        msg: {
          from_user_id: "",
          to_user_id: recipientId,
```

```typescript
/** 上传文件到微信 CDN，返回下载参数 */
async function uploadToCdn(
  account: WechatCredentials,
  filePath: string,
  mediaType: number,
  recipientId: string,
): Promise<{ downloadParam: string; aesKey: Buffer; filesize: number }> {
  const fileBuf = readFileSync(filePath)
  const rawsize = fileBuf.length
  const rawfilemd5 = createHash("md5").update(fileBuf).digest("hex")
  const filesize = aesEcbPaddedSize(rawsize)
  const filekey = randomBytes(16).toString("hex")
  const aesKey = randomBytes(16)

  // 1. 获取上传凭证
  const uploadResp = JSON.parse(await apiFetch(
    "ilink/bot/getuploadurl",
    {
      filekey,
      media_type: mediaType,
      to_user_id: recipientId,
      rawsize,
      rawfilemd5,
      filesize,
      aeskey: aesKey.toString("hex"),
      no_need_thumb: true,
      base_info: { channel_version: CHANNEL_VERSION },
    },
    account.token,
    SEND_TIMEOUT_MS,
  ))

  if (!uploadResp.upload_param) {
    throw new Error("getUploadUrl returned no upload_param")
  }

  // 2. AES 加密后上传到 CDN
  const ciphertext = encryptAesEcb(fileBuf, aesKey)
  const cdnUrl = buildCdnUploadUrl(uploadResp.upload_param, filekey)

  for (let attempt = 1; attempt <= CDN_MAX_RETRIES; attempt++) {
    try {
      const cdnRes = await fetch(cdnUrl, {
        method: "POST",
        headers: { "Content-Type": "application/octet-stream" },
        body: new Uint8Array(ciphertext),
      })
      if (cdnRes.status >= 400 && cdnRes.status < 500) {
        const errMsg = cdnRes.headers.get("x-error-message") ?? (await cdnRes.text())
        throw new Error(`CDN client error ${cdnRes.status}: ${errMsg}`)
      }
      if (cdnRes.status !== 200) {
        const errMsg = cdnRes.headers.get("x-error-message") ?? `status ${cdnRes.status}`
        throw new Error(`CDN server error: ${errMsg}`)
      }
      const downloadParam = cdnRes.headers.get("x-encrypted-param")
      if (!downloadParam) throw new Error("CDN response missing x-encrypted-param")
      return { downloadParam, aesKey, filesize }
    } catch (err: any) {
      if (err.message?.includes("client error")) throw err
      if (attempt >= CDN_MAX_RETRIES) throw err
      log("CDN_UPLOAD_RETRY", `attempt ${attempt}: ${err.message}`)
    }
  }

  throw new Error("CDN upload failed after retries")
}

/** 从 CDN 下载加密媒体 */
async function downloadFromCdn(media: {
  encrypt_query_param?: string
  full_url?: string
  aes_key?: string
}): Promise<Buffer> {
  let cdnUrl: string
  if (media.full_url?.trim()) {
    cdnUrl = media.full_url.trim()
  } else if (media.encrypt_query_param?.trim()) {
    cdnUrl = buildCdnDownloadUrl(media.encrypt_query_param.trim())
  } else {
    throw new Error("Inbound media missing both full_url and encrypt_query_param")
  }

  for (let attempt = 1; attempt <= CDN_MAX_RETRIES; attempt++) {
    try {
      const res = await fetch(cdnUrl, {
        signal: AbortSignal.timeout(CDN_DOWNLOAD_TIMEOUT_MS),
      })
      if (res.status >= 400 && res.status < 500) {
        const errMsg = res.headers.get("x-error-message") ?? (await res.text())
        throw new Error(`CDN client error ${res.status}: ${errMsg}`)
      }
      if (res.status !== 200) throw new Error(`CDN download status ${res.status}`)
      return Buffer.from(await res.arrayBuffer())
    } catch (err: any) {
      if (err.message?.includes("client error")) throw err
      if (attempt >= CDN_MAX_RETRIES) throw err
      log("CDN_DOWNLOAD_RETRY", `attempt ${attempt}: ${err.message}`)
    }
  }

  throw new Error("CDN download failed after retries")
}
```

```typescript
/** WeChat 长轮询主循环（后台单协程，35s HTTP hold） */
function startPollingLoop(
  account: WechatCredentials,
  client: any,
  signal: AbortSignal,
  worktree: string,
) {
  let backoff = 1_000  // 指数退避起始值

  ;(async () => {
    // 注意：前面的分号是必要的，防止上一行未分号结尾的代码与 (async 被拼接
    // 从磁盘恢复状态
    syncBuffer = loadSyncBuffer()
    loadContextTokens()

    log("POLL", "polling loop started")

    while (!signal.aborted) {
      try {
        const { msgs } = await getUpdates(account, LONG_POLL_TIMEOUT_MS, signal)
        backoff = 1_000  // 成功后重置退避
        sendFailCount.clear()  // 轮询成功 = 网络连通，重置所有发送失败计数（图标恢复）

        for (const raw of msgs) {
          await processInboundMessage(raw, account, client, worktree)
        }
      } catch (err: any) {
        if (signal.aborted) break

        // AbortError = 正常超时，继续下一轮
        if (err.name === "AbortError") continue

        // session 过期 -> 需要重登录
        if (err.message?.includes("session timed out")) {
          log("POLL_FATAL", err.message)
          // 通知用户
          try {
            await sendText(account, account.userId, "微信登录已过期，请在终端重新扫码登录")
          } catch { /* best effort */ }
          break
        }

        // context_token stale → 清除 token，等待下条消息携带新 token
        if (err.message?.includes("Context token stale")) {
          log("POLL", "context_token stale, clearing all cached tokens")
          contextTokens.clear()
          saveContextTokens()
          // 继续下一轮轮询，不触发退避
          continue
        }

        // 可重试错误：指数退避
        log("POLL_RETRY", `${err.message}, backoff=${backoff}ms`)
        await sleep(Math.min(backoff, 30_000))
        backoff *= 2
      }
    }

    log("POLL", "polling loop ended")
  })()

  // 插件卸载时清理
  signal.addEventListener("abort", () => {
    log("INFO", "plugin shutting down, polling stopped")
  }, { once: true })
}

/** 处理入站消息：提取 → 去重 → 下载附件 → 注入 OC */
async function processInboundMessage(
  raw: any,
  account: WechatCredentials,
  client: any,
  worktree: string,
) {
  // 只处理用户消息
  if (raw.message_type !== MSG_TYPE_USER) return

  // 提取文本和附件
  const { text, attachments } = extractInboundContent(raw)

  // 跳过空消息
  if (!text && attachments.length === 0) return

  // 去重
  const msgKey = [
    raw.from_user_id ?? "",
    raw.client_id ?? "",
    String(raw.create_time_ms ?? ""),
    raw.context_token ?? "",
  ].join("|")
  if (recentMessageKeys.has(msgKey)) return
  recentMessageKeys.add(msgKey)
  recentMessageOrder.push(msgKey)
  while (recentMessageOrder.length > RECENT_KEYS_MAX) {
    const oldest = recentMessageOrder.shift()!
    recentMessageKeys.delete(oldest)
  }

  const senderId = raw.from_user_id ?? "unknown"
  const sender = senderId.split("@")[0] || senderId

  // 缓存 context_token
  if (raw.context_token) {
    contextTokens.set(senderId, raw.context_token)
    saveContextTokens()
  }

  // 下载附件
  const downloadedPaths: string[] = []
  for (const att of attachments) {
    try {
      const encrypted = await downloadFromCdn(att.media)
      const plaintext = decryptInboundMediaPayload(encrypted, att.aesKey)
      const filePath = saveAttachment(att.fileName || `wechat-${att.kind}`, plaintext)
      downloadedPaths.push(filePath)
      log("ATTACH_DL", `${att.kind}: ${filePath}`)
    } catch (err: any) {
      log("ATTACH_DL_FAIL", `${att.kind}: ${err.message}`)
    }
  }

  // 处理系统指令
  const trimmed = text.trim()
  if (trimmed.startsWith("/")) {
    await handleCommand(trimmed, senderId, account, client, worktree)
    return
  }

  // 首次消息向导：新用户先回复指导语，第二次才创建会话
  if (!wechatSid.has(senderId)) {
    if (await handleFirstContact(text, senderId, account, worktree)) return
  }

  // 同意/拒绝 → 审批
  const approve = trimmed === "同意" || trimmed === "yes" || trimmed === "y"
  const reject = trimmed === "拒绝" || trimmed === "no" || trimmed === "n"
  if (approve || reject) {
    if (resolveLatestPermission(senderId, approve)) {
      await sendText(account, senderId, approve ? "✅ 已批准" : "✅ 已拒绝", undefined, client)
    } else {
      await sendText(account, senderId, "当前没有待审批的请求", undefined, client)
    }
    return
  }

  // 常规消息 → 注入 OC
  log("WX_IN", `[${sender}] ${trimmed.slice(0, 80)}`)
  try {
    const sid = await getOrCreateSession(client, senderId, worktree)
    const prompt = downloadedPaths.length > 0
      ? `${trimmed}\n\n[收到文件] ${downloadedPaths.join(", ")}`
      : trimmed

    await updateSessionIcon(client, sid, "processing")
    await client.session.prompt({
      path: { id: sid },
      body: { parts: [{ type: "text" as any, text: prompt }] },
    })
    log("INJECT", `[${t(sid)}] ← ${trimmed.slice(0, 60)}`)
  } catch (err: any) {
    log("INJECT_FAIL", `${err.message}`)
    try {
      await sendText(account, senderId, "消息已收到，但注入会话失败，请稍后重试", undefined, client)
    } catch { /* best effort */ }
  }
}

/** 从原始消息中提取文本和附件描述 */
function extractInboundContent(raw: any): {
  text: string
  attachments: Array<{ kind: string; fileName: string; media: any; aesKey: string }>
} {
  const lines: string[] = []
  const attachments: Array<{ kind: string; fileName: string; media: any; aesKey: string }> = []

  for (const item of raw.item_list ?? []) {
    // 引用消息
    if (item.ref_msg) {
      const refParts: string[] = []
      if (item.ref_msg.title?.trim()) refParts.push(item.ref_msg.title.trim())
      if (item.ref_msg.message_item?.text_item?.text?.trim()) {
        refParts.push(item.ref_msg.message_item.text_item.text.trim())
      }
      if (refParts.length) lines.push(`引用: ${refParts.join(" | ")}`)
    }

    if (item.type === MSG_ITEM_TEXT) {
      const t = item.text_item?.text?.trim()
      if (t) lines.push(t)
    }

    if (item.type === MSG_ITEM_VOICE) {
      const transcript = item.voice_item?.text?.trim()
      if (transcript) lines.push(transcript)
    }

    if (item.type === MSG_ITEM_IMAGE) {
      const media = item.image_item?.media
      const aesKey = media?.aes_key ?? media?.aeskey ?? item.image_item?.aes_key ?? item.image_item?.aeskey
      if (media && aesKey?.trim()) {
        attachments.push({
          kind: "image",
          fileName: item.image_item?.file_name?.trim() || "wechat-image.jpg",
          media,
          aesKey,
        })
      } else {
        lines.push("[微信图片: 无法下载]")
      }
    }

    if (item.type === MSG_ITEM_FILE) {
      const media = item.file_item?.media
      const aesKey = media?.aes_key ?? media?.aeskey ?? item.file_item?.aes_key ?? item.file_item?.aeskey
      if (media && aesKey?.trim()) {
        attachments.push({
          kind: "file",
          fileName: item.file_item?.file_name?.trim() || "wechat-file",
          media,
          aesKey,
        })
      } else {
        lines.push("[微信文件: 无法下载]")
      }
    }
  }

  return { text: lines.join("\n").trim(), attachments }
}

/** 保存附件到磁盘 */
function saveAttachment(fileName: string, data: Buffer): string {
  const today = new Date().toISOString().slice(0, 10)
  const dir = join(ATTACHMENTS_DIR, today)
  mkdirSync(dir, { recursive: true })
  const safeName = fileName
    .replace(/[<>:"/\\|?*]+/g, "_")
    .replace(/\s+/g, " ")
    .trim()
    .slice(0, 160)
  const uniqueName = `${today}-${randomBytes(4).toString("hex")}-${safeName}`
  const fp = join(dir, uniqueName)
  writeFileSync(fp, data)
  return fp
}
```

```typescript
/**

 * | `/confirm <code>` | `/批准 <code>` | 批准权限请求（不指定 code 则批准最新） |

function resolveDir(dirIdx: number | null, worktree: string): string {
  if (dirIdx === null || dirIdx < 1 || dirIdx > _projectDirs.length) return worktree
  return _projectDirs[dirIdx - 1]
}

function getNick(dir: string): string {
  const nick = basename(dir)
  return nick || dir
}

/** 查询全部分组的顶层会话，返回 flat list + 每个 session 附 dirIndex/dirName */
async function listAllSessions(client: any): Promise<{ flat: Session[]; dirMap: Map<string, number> }> {
  const flat: Session[] = []
  const dirMap = new Map<string, number>() // sessionID → directory index
  for (let di = 0; di < _projectDirs.length; di++) {
    try {
      const resp: any = await client.session.list({ directory: _projectDirs[di] })
      const all: Session[] = Array.isArray(resp) ? resp : resp.data ?? []
      for (const s of all) {
        if (!s.parentID) {
          flat.push(s)
          globalIdx++
          dirMap.set(s.id, di)
        }
      }
    } catch { /* skip unreachable dirs */ }
  }
  return { flat, dirMap }
}

/** 处理微信端发来的指令（以 / 开头） */
async function handleCommand(
  cmd: string,
  senderId: string,
  account: WechatCredentials,
  client: any,
  worktree: string,
) {
  const parts = cmd.slice(1).split(/\s+/)
  let command = parts[0].toLowerCase()
  const args = parts.slice(1)

  // 兼容无空格参数：/join2 → command="join" args=["2"]
  const attached = command.match(/^(join|bind|switch|new|unbind)(\d+)$/)
  if (attached) {
    command = attached[1]
    args.unshift(attached[2])
  }

  const wx = (text: string) => sendText(account, senderId, text, undefined, client)

  switch (command) {
    // -------- 中断 --------
    case "stop":
    case "停止": {
      log("CMD", "stop")
      const sid = wechatSid.get(senderId)
      if (sid) {
        try { await client.session.abort({ path: { id: sid } }) } catch { /* ok */ }
      }
      await wx("✅ 已中断当前任务")
      break
    }

    // -------- 状态 --------
    case "status":
    case "状态": {
      const currentSid = wechatSid.get(senderId)
      try {
        const { flat } = await listAllSessions(client)
        const sessionLines = flat.map((s, i) => {
          const marker = s.id === currentSid ? " [当前会话]" : ""
          return `${i + 1}. ${s.title}${marker}`
        })
        const lines = [
          `当前目录：${getNick(_projectDirs[0] ?? "")}`,
          `会话（${flat.length} 个）：`,
          ...(sessionLines.length ? sessionLines : ["  (无会话)"]),
        ]
        await wx(lines.join("\n"))
      } catch {
        await wx("获取状态失败")
      }
      break
    }

    // -------- 进入目录 --------
    case "join":
    case "进入": {
      const n = parseInt(args[0], 10)
      if (isNaN(n) || n < 1 || n > _projectDirs.length) {
        const list = _projectDirs.map((d, i) => `${i + 1}. ${getNick(d)}`).join("\n")
        await wx(`请指定目录编号（1-${_projectDirs.length}）：\n${list}`)
        break
      }
      _joinDir = n
      if (_joinDirTimer) clearTimeout(_joinDirTimer)
      _joinDirTimer = setTimeout(() => { _joinDir = null; _joinDirTimer = null }, 5 * 60 * 1000)
      await wx(`已进入 📁 ${getNick(_projectDirs[n - 1])}\n回复 /new 新建 /sessions 查看 /bind <编号> 绑定`)
      break
    }

    // -------- 绑定会话 --------
    case "bind":
    case "绑定": {
      const idx = parseInt(args[0], 10)
      if (isNaN(idx)) {
        await wx("请指定会话编号，例如 /bind 2")
        break
      }
      const { flat } = await listAllSessions(client)
      if (idx < 1 || idx > flat.length) {
        await wx(`❌ 编号超出范围（1-${flat.length}）`)
        break
      }
      const matched = flat[idx - 1]
      wechatSid.set(senderId, matched.id)
      sidTitle.set(matched.id, matched.title)
      saveSessionMapping(worktree)
      await updateSessionIcon(client, matched.id, "normal")
      if (_joinDirTimer) { clearTimeout(_joinDirTimer); _joinDirTimer = null }
      _joinDir = null
      await wx(`✅ 已绑定会话: ${matched.title} (${matched.id.slice(0, 12)}…)`)
      break
    }

    // -------- 新建 --------
    case "new":
    case "新建": {
      log("CMD", "new-session")
      const dirIdx = args.length > 0 ? parseInt(args[0], 10) : (_joinDir ?? null)
      const targetDir = resolveDir(dirIdx, worktree)
      try {
        const title = wechatTitle(senderId)
        const resp: any = await client.session.create({
          directory: targetDir,
          body: { title },
        })
        const newSid = resp.id ?? resp.sessionID ?? resp.data?.id
        if (newSid) {
          wechatSid.set(senderId, newSid)
          sidTitle.set(newSid, title)
          saveSessionMapping(worktree)
          await updateSessionIcon(client, newSid, "normal")
          if (_joinDirTimer) { clearTimeout(_joinDirTimer); _joinDirTimer = null }
          _joinDir = null
          await wx(`✅ 已创建新会话 [${t(newSid)}]`)
        }
      } catch (err: any) {
        log("CMD_NEW_FAIL", `${err}`)
        await wx("❌ 创建会话失败")
      }
      break
    }

    // -------- 列出会话（多目录分组） --------
    case "sessions":
    case "会话": {
      log("CMD", "sessions")
      try {
        const { flat, dirMap } = await listAllSessions(client)
        const currentSid = wechatSid.get(senderId)

        if (_projectDirs.length === 0) {
          await wx("暂无项目目录")
          break
        }

        const lines: string[] = []
        let globalStart = 0

        if (_projectDirs.length === 1) {
          lines.push(`📁 ${getNick(_projectDirs[0])} (${flat.length}个):`)
          for (let i = 0; i < flat.length; i++) {
            const marker = flat[i].id === currentSid ? " ← 当前" : ""
            lines.push(`  ${i + 1}. ${flat[i].title.slice(0, 24)} (${flat[i].id.slice(0, 8)}…)${marker}`)
          }
        } else {
          for (let di = 0; di < _projectDirs.length; di++) {
            const dirSessions = flat.filter((s) => dirMap.get(s.id) === di)
            if (dirSessions.length === 0) continue
            lines.push(`📁 ${getNick(_projectDirs[di])} (${dirSessions.length}个):`)
            for (let i = 0; i < dirSessions.length; i++) {
              const globalIdx = globalStart + i + 1
              const marker = dirSessions[i].id === currentSid ? " ← 当前" : ""
              lines.push(`  ${globalIdx}. ${dirSessions[i].title.slice(0, 24)} (${dirSessions[i].id.slice(0, 8)}…)${marker}`)
            }
            globalStart += dirSessions.length
          }
        }
        lines.push("回复 /switch <编号> 或 /join <编号> 切换目录")
        await wx(lines.join("\n"))
      } catch (err: any) {
        log("CMD_SESSIONS_FAIL", `${err}`)
        await wx("❌ 获取会话列表失败")
      }
      break
    }

    // -------- 切换会话（跨目录） --------
    case "switch":
    case "切换": {
      log("CMD", `switch ${args[0] ?? ""}`)
      const target = args.join(" ").trim()
      if (!target) {
        await wx("请指定会话编号或 ID，例如 /switch 2 或 /switch abc12345")
        break
      }
      try {
        const { flat } = await listAllSessions(client)
        let matched: Session | null = null

        const n = parseInt(target, 10)
        if (n >= 1 && n <= flat.length) {
          matched = flat[n - 1]
        } else {
          matched = flat.find(s => s.id.startsWith(target)) ?? null
        }

        if (!matched) {
          await wx(`❌ 未找到匹配的会话: ${target}`)
          break
        }
        const prevSid = wechatSid.get(senderId)
        wechatSid.set(senderId, matched.id)
        sidTitle.set(matched.id, matched.title)
        saveSessionMapping(worktree)
        if (prevSid && prevSid !== matched.id) {
          await updateSessionIcon(client, prevSid, "normal")
        }

        await wx("❌ 切换会话失败")
      }
      break
    }

    // -------- 批准 --------
    case "confirm":
    case "批准": {

      if (!code) {
        await wx("请指定审批码，例如 /confirm 3A8F")
        break
      }
      if (resolvePermission(code, senderId, true)) {
        await wx(`✅ 已批准 #${code}`)
      } else {
        await wx(`❌ 未找到审批码 #${code}`)
      }
      break
    }

    // -------- 拒绝 --------
    case "deny":
    case "拒绝": {
      const code = args[0]
      if (!code) {
        await wx("请指定审批码，例如 /deny 3A8F")
        break
      }
      if (resolvePermission(code, senderId, false)) {
        await wx(`✅ 已拒绝 #${code}`)
      } else {
        await wx(`❌ 未找到审批码 #${code}`)
      }
      break
    }

    // -------- 解绑 --------
    case "unbind":
    case "解绑": {
      log("CMD", "unbind")
      const oldSid = wechatSid.get(senderId)
      if (!oldSid) {
        await wx("当前未绑定任何会话")
        break
      }
      wechatSid.delete(senderId)
      saveSessionMapping(worktree)
      await updateSessionIcon(client, oldSid, "normal")
      await wx("✅ 已解绑，后续消息将创建新会话")
      break
    }

    // -------- 重命名 --------
    case "rename":
    case "改名": {
      log("CMD", `rename ${args.join(" ").slice(0, 40)}`)
      const newName = args.join(" ").trim()
      if (!newName) {
        await wx("请指定新标题，例如 /rename 我的会话")
        break
      }
      const sid = wechatSid.get(senderId)
      if (!sid) {
        await wx("当前未绑定会话")
        break
      }
      const oldTitle = sidTitle.get(sid) ?? ""
      const base = stripIconPrefix(oldTitle)
      try {
        await client.session.update({ path: { id: sid }, body: { title: newName } })
        sidTitle.set(sid, `${WECHAT_ICON}${newName}`)
        log("SESSION", `[${base}] → [${newName}]`)
        await wx(`✅ 已重命名: ${newName}`)
      } catch (err: any) {
        log("CMD_RENAME_FAIL", `${err}`)
        await wx("❌ 重命名失败")
      }
      break
    }

    // -------- 帮助 --------
    case "help":
    case "帮助": {
      await wx(`指令列表：
/stop /停止  中断 AI 任务
/status /状态  当前状态
/join N /进入 N  进入目录
/bind M /绑定 M  绑定会话
/unbind /解绑  解绑会话
/rename /改名  修改标题

/new /新建  创建会话
/sessions /会话  列出会话
/switch N /切换 N  切换会话

      break
    }

    default:
      await wx(`未知指令: /${command}\n支持: /stop /停止, /status /状态, /join /进入, /bind /绑定, /unbind /解绑, /rename /改名, /new /新建 [N], /sessions /会话, /switch /切换, /confirm /批准, /deny /拒绝`)
  }
}
```

### 3.13 Section 12: Event Hooks

```typescript
// ============================================================
// Section 12: Event Hooks
// ============================================================

function createEventHandler(client: any) {
  return async ({ event }: { event: any }) => {
    // 后台登录未完成时跳过（此时 _creds 为 null）
    if (!_creds) return

    // ---- session.idle: AI 完成 → 发最终回复到微信 ----
    // 注意：handler 参数是 { event }，不是 Event 流式订阅，无 directory/workspace 字段
    // 此 handler 中的 client 是插件入口注入的 client 对象，可直接调用 session.messages()
    if (event.type === "session.idle") {
      const ev = event as EventSessionIdle
      const sid = ev.properties.sessionID
      log("IDLE", `[${t(sid)}] AI completed`)

      const wechatId = findWechatSender(sid)
      if (!wechatId) return

      // 恢复会话标题为正常状态（离开"处理中"）
      await updateSessionIcon(client, sid, "normal")

      // 获取最终回复（通过 client.session.messages 查询最近一条 assistant 消息）
      // 注意：EventSessionIdle.properties 只包含 sessionID，没有 finalText/text/messages
      let finalText = ""
      try {
        const msgsResp: any = await client.session.messages({ path: { id: sid }, query: { limit: 5 } })
        const msgs = Array.isArray(msgsResp) ? msgsResp : msgsResp.data ?? []
        // 从后往前找最近的一条 assistant 消息
        for (let i = msgs.length - 1; i >= 0; i--) {
          const msg = msgs[i]
          if (msg.info?.role === "assistant") {
            const parts = Array.isArray(msg.parts) ? msg.parts : []
            finalText = parts
              .filter((p: any) => p.type === "text" && p.ignored !== true)
              .map((p: any) => p.text ?? "")
              .join("")
              .trim()
            if (finalText) break
          }
        }
      } catch (err) {
        log("IDLE_GET_MSG_FAIL", `${err}`)
      }
      if (!finalText) return

      try {
        await sendText(_creds!, wechatId, finalText.slice(0, 2000), undefined, client)
        log("WX_OUT", `[${t(sid)}] → ${finalText.slice(0, 80)}`)
      } catch (err: any) {
        log("WX_OUT_FAIL", `[${t(sid)}] ${err.message}`)
        recordSendResult(sid, false, client)
      }
      return
    }

    // ---- session.created: 更新标题缓存 ----
    if (event.type === "session.created") {
      const ev = event as EventSessionCreated
      const s = ev.properties.info
      if (!s.parentID) {
        sidTitle.set(s.id, s.title)
        log("SESSION", `[${t(s.id)}] created`)
      }
      return
    }

    // ---- session.deleted: 清理缓存 ----
    if (event.type === "session.deleted") {
      const ev = event as EventSessionDeleted
      const s = ev.properties.info
      sidTitle.delete(s.id)
      for (const [wx, sid] of wechatSid) {
        if (sid === s.id) { wechatSid.delete(wx); break }
      }
      log("SESSION", `[${t(s.id)}] deleted`)
      return
    }

    // ---- session.updated: 更新标题 ----
    if (event.type === "session.updated") {
      const ev: any = event
      const s: Session = ev.properties.info
      if (!s.parentID && sidTitle.has(s.id) && sidTitle.get(s.id) !== s.title) {
        const old = sidTitle.get(s.id)!
        sidTitle.set(s.id, s.title)
        log("SESSION", `[${old}] → [${s.title}]`)
      }
      return
    }

    // ---- message.part.updated: 流式输出 ----
    // 可用事件（v1 SDK Event 联合类型已包含）：
    //   type: "message.part.updated"
    //   properties: { part: Part, delta?: string }
    // Part 包含: { id, sessionID, messageID, type, text, ignored? }
    //
    // 设计取舍：当前版本不做流式转发到微信（理由见下文），
    // 保留此 handler 占位，方便后续修改。
    // 若启用流式转发，可在此累加 delta 到 per-part 缓存，
    // 在 session.idle 时一并发出。
    if (event.type === "message.part.updated") {
      return
    }
  }
}
```

**流式转发设计取舍**：

| 方案 | 优点 | 缺点 |
|:----|:-----|:-----|
| **仅最终回复**（当前选择） | 实现简单，微信端不被打断 | 用户等待时不知道进度 |
| **增量流式转发** | 微信端实时看到 AI 输出 | 消息碎片多，微信通知频繁；部分内容为思考过程/非最终结论 |
| **定期汇总推送**（每 5-10s） | 折中方案 | 实现复杂度高 |

**选用"仅最终回复"的原因**：
1. 原始 bridge 也是基于 `final_reply` 事件转发最终回复（见 `wechat-bridge.js` 中的 `final_reply` handler）
2. 微信的 `sendmessage` API 每次调用都会触发用户通知，频繁推送影响体验
3. 如需进度反馈，可后续通过 `wechat_status` 工具或定期 "AI 仍在处理..." 通知实现

### 3.14 Section 12b: Permission Handler

> **实际 API 确认**（来自 `@opencode-ai/plugin` 类型定义）：
> - 钩子名：`"permission.ask"`（字符串 key，非 `permission.asked`）
> - 签名：`(input: Permission, output: { status: "ask" | "deny" | "allow" }) => Promise<void>`
> - 通过修改 `output.status` 来响应，非返回值
> - `Permission` 包含：`{ id, type, pattern?, sessionID, messageID, callID?, title, metadata, time }`
> - 异步支持：handler 返回 Promise，OC 会等待；设 `"allow"` 允许，`"deny"` 拒绝，`"ask"` 弹终端确认框

```typescript
// ============================================================
// Section 12b: Permission Handler
// ============================================================

// ---- 审批队列（模块级状态） ----
interface PendingPermission {
  code: string
  senderId: string
  sessionID: string
  permissionID: string
  resolve: (value: boolean) => void
}
const pendingPermissions: PendingPermission[] = []

/**
 * 创建 permission.ask 钩子
 * 
 * 工作流程：
 * 1. 检查当前 session 是否对应用户的 WeChat sender
 * 2. 如果不是 WeChat 发起的 → output.status = "ask"（放行给终端）
 * 3. 如果是 WeChat 发起的 → 发微信审批请求 → await 用户回复 → 设 status
 * 
 * 注意：此 handler 会异步阻塞，直到用户通过微信回复或超时。
 * 5 分钟无回复自动拒绝。
 *
 * 【UI 定制路径】如果 OC UI 层可定制，审批流程可改用非阻塞方案：
 *   1. 在 permission.ask 中设 output.status = "allow"（不对 WeChat 任务弹窗）
 *   2. 通过 client.tui.showToast() 或自定义事件将审批通知推送到微信
 *   3. 微信用户确认后，通过 client.session 的 SDK 权限 API 远程审批
 *   这样不阻塞 OC 主线程，体验更流畅。当前版本先用阻塞方案，后续可切换。
 */
function createPermissionHandler(client: any) {
  return async (input: any, output: { status: "ask" | "deny" | "allow" }) => {
    // 后台登录未完成时跳过审批（直接 ask，让 OC 内置审批流程处理）
    if (!_creds) { output.status = "ask"; return }
    const sessionID = input.sessionID
    const permissionID = input.id
    const toolName = input.type ?? ""

    // 找到对应的 wechat sender
    const wechatId = findWechatSender(sessionID)
    if (!wechatId) {
      // 不是 WeChat 发起的任务，放行给终端弹窗
      output.status = "ask"
      return
    }

    const code = buildOneTimeCode()
    log("PERM", `#${code} tool=${toolName} session=${sessionID.slice(0, 16)}`)

    // 1. 先发送审批请求到微信（必须在等待之前）
    try {
      const desc = input.metadata?.command ?? input.title ?? `工具: ${toolName}`
      await sendText(_creds!, wechatId,
        `[审批 #${code}] 需要确认：${desc.slice(0, 200)}\n` +
        `/confirm ${code} 或 /deny ${code}`,
        undefined, client,
      )
      log("PERM_SENT", `#${code} to ${wechatId.slice(0, 16)}`)
    } catch (err: any) {
      log("PERM_SEND_FAIL", `#${code}: ${err.message}`)
      output.status = "ask"  // 发送失败，回退到终端弹窗
      return
    }

    // 2. 再等待用户回复（异步阻塞，最长 5 分钟）
    const approved = await new Promise<boolean>((resolve) => {
      pendingPermissions.push({
        code,
        senderId: wechatId,
        sessionID,
        permissionID,
        resolve,
      })

      setTimeout(() => {
        const idx = pendingPermissions.findIndex(p => p.code === code)
        if (idx >= 0) {
          pendingPermissions.splice(idx, 1)
          resolve(false)
        }
      }, 5 * 60 * 1000)
    })

    // 3. 根据用户回复设置 status
    output.status = approved ? "allow" : "deny"
    log("PERM", `#${code} → ${output.status}`)

    if (!approved) {
      try {
        await sendText(_creds!, wechatId, `⏰ 审批 #${code} 已超时自动拒绝`, undefined, client)
      } catch { /* best effort */ }
    }
  }
}

/** 查找 pending 审批 */
function findPendingPermission(code: string, senderId: string): PendingPermission | null {
  return pendingPermissions.find(p => p.code === code && p.senderId === senderId) ?? null
}

/** 解析审批 */
function resolvePermission(code: string, senderId: string, approved: boolean): boolean {
  const perm = findPendingPermission(code, senderId)
  if (!perm) return false
  const idx = pendingPermissions.indexOf(perm)
  if (idx >= 0) pendingPermissions.splice(idx, 1)
  perm.resolve(approved)
  return true
}
```

## 5. 错误处理参考

### 5.1 错误分类与处理策略

| 错误场景 | 表现 | 处理 |
|---------|------|------|
| 长轮询超时 (35s) | AbortError | 继续下一轮（正常流程） |
| 网络断开 | fetch 抛 NetworkError | 指数退避重试：1s → 2s → 4s → ... → 30s |
| HTTP 5xx / 429 | fetch 抛 HTTP status 错误 | 指数退避重试 |
| session timeout (errcode=-14) | apiFetch 抛 "WeChat session timed out" | 断轮询 + 报错要求重登录 |
| context_token stale (ret=-2) | apiFetch 抛 "Context token stale" | 清除所有缓存 token，继续轮询等待新 token |
| 401/403 | HTTP 401/403 | 凭据过期 → 触发重登录 |
| CDN upload 4xx | "CDN client error" | 不重试（客户端错误） |
| CDN upload 5xx | "CDN server error" | 重试最多 3 次 |
| CDN download 4xx | 同上 | 不重试 |

### 5.2 重试逻辑说明

CDN 上传/下载自带内联重试（见 Section 8，最多 3 次，4xx 不重试）。长轮询失败使用指数退避（见 Section 9，1s → 2s → ... → 30s）。以下为通用的重试模板，仅供参考，未在插件代码中使用：

```typescript
async function withRetry<T>(
  fn: () => Promise<T>,
  options: { maxRetries?: number; isFatal?: (err: Error) => boolean } = {},
): Promise<T> {
  const maxRetries = options.maxRetries ?? 3
  const isFatal = options.isFatal ?? (() => false)

  for (let attempt = 1; attempt <= maxRetries; attempt++) {
    try {
      return await fn()
    } catch (err: any) {
      if (isFatal(err) || attempt >= maxRetries) throw err
      log("RETRY", `attempt ${attempt}/${maxRetries}: ${err.message}`)
      await sleep(Math.min(1_000 * Math.pow(2, attempt - 1), 30_000))
    }
  }

  throw new Error("Unexpected: loop ended without success or throw")
}
```

### 5.3 插件卸载清理

当 OpenCode 卸载插件时，`abortController.abort()` 被触发：
- `startPollingLoop` 的 `while (!signal.aborted)` 退出
- 正在等待的 `getUpdates` 因 `signal.aborted` 抛 AbortError → 被 `catch` 捕获 → `signal.aborted` 为 true → 退出循环
- 所有 pending 审批自动拒绝（5 分钟超时）

### 5.4 性能保障

设计上保证插件对 OC 和电脑零影响：

| 资源 | 表现 | 说明 |
|:----|:-----|:-----|
| **CPU** | 空闲时 ≈ 0% | 唯一活动是 35s HTTP 长轮询 `await`，不占用 CPU 时间片 |
| **内存** | 恒定 ~10MB | 仅缓存 session 映射、context_token、去重 key（均限制上限） |
| **网络** | 1 条 TCP 连接 | 长轮询保持单连接，35s 超时后重建，无并发 |
| **磁盘 I/O** | 仅日志 + 状态持久化 | 日志单行追加（`appendFileSync`），状态文件仅在映射变更时写入 |
| **OC 主线程** | 完全不阻塞 | 轮询在异步协程中运行；`permission.ask` 是唯一可能阻塞的点（待验证），但只影响 WeChat 发起的任务 |
| **启动速度** | 不影响 OC 启动 | 插件入口是 async 函数，二维码登录失败才 throw；session 缓存延迟初始化（`setTimeout(0)`）|

**关键设计决策**：
- 不用定时器轮询（CPU 空转），用 HTTP 长轮询（事件驱动）
- 所有 I/O 用异步非阻塞（`await fetch`、`await client.*`）
- 文件写入用同步 API（`writeFileSync`、`appendFileSync`）但数据量极小（< 1KB），不影响性能
- 无子进程、无 IPC、无 socket 监听

## 7. 常见问题

### Q: 为什么零依赖？
A: `node:crypto` 提供 AES-128-ECB，全局 `fetch` 提供 HTTP，二维码用 `api.qrserver.com` 在线生成 + HTML 文件。这样 `npm install` 只需安装 peer deps，安装速度极快。

### Q: 与原 bridge.exe 凭据兼容吗？
A: 兼容。v2.5 起数据目录改为 `~/.opencode/wechat-bridge/`（跨项目共享）。首次启动自动检测旧 `~/.cli-bridge/` 目录并迁移到新位置。

### Q: 如何调试？
A: 查看 `<插件目录>/log/wechat-bridge.log`。设置 `WECHAT_OPENCODE_DEBUG=1` 环境变量可开启更详细日志。

### Q: session 如何映射？
A: 每个微信用户绑定一个 OC session。首次发消息时自动创建，之后复用。映射持久化到 `{worktree}/.opencode/plugins/wechat-bridge/session-map.json`，插件重启后自动恢复。用户在本地终端创建新 session 后，微信消息仍发送到原 session（通过 `/new` 指令手动切换）。

### Q: 支持多人同时使用吗？
A: 当前版本不支持。插件绑定到一个微信 Bot 账户，只有该 Bot 的消息会被接收。多用户场景可在未来版本通过区分 `senderId` 实现。

### Q: 文件大小限制？
A: 接收图片 ≤ 20MB，文件 ≤ 50MB。可通过 `WECHAT_MAX_INBOUND_IMAGE_MB` 和 `WECHAT_MAX_INBOUND_FILE_MB` 环境变量调整。

### 9.6 UI 定制说明

OC 的 UI 层是可定制的，这为微信桥接插件提供了丰富的增强路径。以下对比"插件内自闭环"与"UI 定制增强"两种方案：

#### 状态指示灯（微信图标 × 状态前缀）

标题前缀在 OC TUI 会话列表中就是状态灯，无需额外 UI 组件：

| 图标 | 含义 | 设置时机 |
|:----:|:-----|:---------|
| `📱` | 正常 | 发送成功、轮询连通、重新发消息时 |
| `💬` | AI 处理中 | `client.session.prompt()` 注入后 → `session.idle` 收到前 |
| `📵` | 发送降级 | `sendText`/`sendImage` 连续失败 ≥ 3 次 |
| `🔴` | 登录过期 | polling 捕获 session timeout |

#### 完整功能对比

| 场景 | 传统方案 | UI 定制方案 |
|:----|:---------|:------------|
| **权限审批** | `permission.ask` 异步阻塞等待微信回复 | 设 `output.status = "allow"` 跳过弹窗，通过推送通知到微信做二次确认，用 SDK `postSessionIdPermissionsPermissionId` API 远程审批 |
| **状态指示灯** | 无 | 会话标题前缀（`📱` `💬` `📵` `🔴`）在 TUI 中实时可见 |
| **Toast 通知** | 仅通过微信消息回复 | 监听 `tui.toast.show` 事件在消息到达时弹 Toast（插件 `client` 无可调用方法）|
| **状态栏信息** | 仅通过微信 `/status` 查询 | 无 `client.tui.setStatus()` API，如需要可考虑通过 `tui.command.execute` 事件注册自定义命令 |
| **OC 内命令** | 无 | 注册 `wechat:status` `wechat:reconnect` 命令，OC 命令行中可管理 |
| **会话切换** | 微信 `/new` 指令 | 监听 `tui.command.execute` 事件处理自定义会话切换命令 |
| **审批计数提示** | 无 | 会话标题附加值如 `📱微信-abc123 [2]`，括号内显示待审批数 |

#### 图标状态的自动恢复链路

```
sendText/sendImage 失败
  → sendFailCount[sid] += 1
  → 连续失败 >= 3 → updateSessionIcon(sid, "degraded") → 标题变 📵
  → 后续轮询成功收到消息 → sendFailCount.clear()（所有 sid 恢复 📱）
  → 或下一次 send 成功 → recordSendResult(sid, true) → 该 sid 恢复 📱
  → 连续失败到达上限但 polling 也失败 → 最终状态与网络一致
```

#### 终端仿真兼容性

| 图标 | Unicode | 常见终端 |
|:----:|:--------|:---------|
| `📱` | U+1F4F1 | ✅ 全部支持 |
| `💬` | U+1F4AC | ✅ 全部支持 |
| `📵` | U+1F4F5 | ✅ 大部分现代终端 |
| `🔴` | U+1F534 | ✅ 全部支持 |

如果终端不支持 `📵`，可降级为 `📱[!]`（文本形式）。

> **注意**：`client.tui.showToast()` / `client.tui.executeCommand()` / `client.tui.appendPrompt()` / `client.tui.publish()` **在插件 client 上可用**（`typeof=function`）。这些方法也可以通过监听 `tui.toast.show` / `tui.command.execute` / `tui.prompt.append` 事件来响应。两者可以共存。

### B.3 模式切换（/mode 设置）

**结论：不支持。** OC 没有服务端 API 可以切换 session mode。原因：

- `client.session.update({ body: { mode } })` 的 `SessionUpdateData` 只允许 body 包含 `title` 字段，`mode` 被服务端忽略
- `client.tui.executeCommand()` 只处理 TUI 系统命令（`session.list`, `agent.cycle` 等），不处理 slash 命令
- `client.session.command()` 会创建一个 AI 回复（导致 OC 播放通知音），且 `/mode` 参数被当作 AI prompt 处理，不改变实际 mode
- `/mode` 是 OC **客户端 slash 命令**，只能在 TUI 里通过用户输入执行，没有 HTTP API 等价

**当前行为：** `/mode` 只读不写。查询时从 `session.messages()` 的最新 `AssistantMessage.mode` 字段读取真实 mode，无消息时回退到 `_modeCache`，再回退到 `"build"`。

**未来可能：** 如果 OC 开放了服务端设置 mode 的 API，可以恢复此功能。
