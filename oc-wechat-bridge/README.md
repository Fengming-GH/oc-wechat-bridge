# oc-wechat-bridge

微信 ↔ OpenCode 双向桥接插件。在微信中直接与 OC AI 对话，接收实时进度、切换会话、审批权限。

## 功能

- **双向消息**：微信↔OC 消息实时互通，AI 回复自动转发到微信
- **实时进度**：AI 思考中、工具调用、文字输出实时推送，不等 AI 完成
- **多项目目录**：自动发现同级带 `.opencode/` 的项目，会话按目录分组显示
- **会话管理**：全局编号 `/switch` 跨目录切换、`/status` 查看全貌

- **自动恢复**：AI 出错或上下文压缩后自动重试，微信端无感
- **跨会话转发**：AI 可通过 `！` 指令将消息转发到其他会话
- **消息不阻塞**：`promptAsync` 注射消息后立即返回，AI 处理期间仍可接收新消息

## 安装

将 `src/index.ts` 复制到 OC 项目的 `.opencode/plugins/` 目录下，文件名任意：

```bash
cp oc-wechat-bridge/src/index.ts 你的项目/.opencode/plugins/wechat-bridge.ts
```

重启 OC，首次启动弹出二维码，微信扫码登录。

无需 `npm install`，插件零外部运行时依赖。

## 指令

所有指令通过微信发送给 Bot，以 `/` 开头。中英文别名均可识别。

| 指令 | 别名 | 功能 |
|------|------|------|
| `/stop` | `/停止` | 中断 AI 任务 |
| `/status` | `/状态` / `/会话` | 查看所有目录和会话 |
| `/switch N` | `/切换 N` | 切换到指定会话并绑定 |
| `/new [N]` | `/新建 [N]` | 创建新会话 |
| `/unbind` | `/解绑` | 解绑当前会话 |
| `/rename <标题>` | `/改名 <标题>` | 修改会话标题 |
| `/mode` | `/模式` | 查看当前会话模式（只读） |
| `/help` | `/帮助` | 显示指令列表 |

**无空格参数：** `/switch3` 与 `/switch 3` 等价。

## AI 工具 / 跨会话指令

AI 对话中支持以下操作：

| 输入 | 说明 |
|------|------|
| `！会话` 或 `!sessions` | AI 调用 `list_sessions` 列出所有会话 |
| `！<前缀> <消息>` 或 `!<前缀> <消息>` | AI 调用 `forward_to_session` 转发消息到目标会话 |

`！`（全角）和 `!`（半角）都支持。

## 进度与格式

AI 处理时微信实时收到：

```
思考中...       ← 推理进度
read            ← 工具调用
edit
好的……         ← AI 回复文字
```

会话列表按目录分组，带空行分隔：

```
📁 GitHub-OCPlugins — 3 个会话
  1. 📱WeChat插件 [当前会话]
  2. 微信测试
  3. 改造计划审查

📁 BT-Valve — 1 个会话
  4. 编译 4.1 版本
```

## 技术栈

- 单文件 TypeScript（~740 行）
- 零外部运行时依赖（仅 OC SDK）
- 原生 Web Fetch API
- AES-128-ECB 加密传输（CDN 媒体文件）
- 事件驱动架构（`message.part.updated` / `session.idle` 等）
